This inventory management system contains a matrix of all chemical recipes (codified for NDA purposes). With this information, and using JavaScript, production orders can be created that help manage the components used, the amount needed to create a certain batch of product, among others. The explanation for each spreadsheet tab can be found below:

  • Administraci칩n: Used for top-level management purposes, especially to keep track of the phase in which production order is in and the essential information (product, amount, etc.). With the checkboxes, the manager can authorize the next phase to begin (production, shipping, etc.)

  • Producci칩n: Used to visualize order priority and, when selecting a specific order, to visualize the exact amount of ingredients needed to create and satisfy the final product needs. It utilizes the SORTN function with specific filters to only show those orders in production and to order them by priority level.

  • Consumo Final: Displays all relevant consumption information, including total consumption numbers, consumption within a specific date range and the top/bottom 10 components used in total and in the set date range.

  • Producci칩n Final: Displays all relevant production information, including total production numbers, production within a specific date range and the top/bottom 10 products created in total and in the set date range.

  • Base de Datos: Contains all the raw, historic production data, including every specific component needed to satisfy every order. Used mainly as the pool of information for the above tabs.

  • Calc Individual: Mainly used for creating new production orders. The dark-blue cells show those that should be changed every time a new order is created. There is a conditional check that verifies no two orders may share order number as well as a display of product name and number for verification purposes. The button makes use of the Google Script attached below to add the production order to the database.

  • Matriz Recetas: Contains the raw, codified recipes matrix with all the final products and the percentage of each component it requires for proper production.

  • Glosario: A tab that contains all component/product names and units for decodification purposes.

  • C치lculo: Used for calculating the numbers that the production orders utilize and display, mainly the specific, practical quantity for each component and the total waste-product expected.

        function agregarDatos() {
          var spreadsheet = SpreadsheetApp.getActive();
          spreadsheet.getRange('B14:N22').activate();
          var bd = SpreadsheetApp.getActiveSpreadsheet().getSheetByName('Base de Datos');
          var lastrow = bd.getLastRow();
          bd.getRange(lastrow+1,1).activate();
          spreadsheet.getRange('\'Calc Individual\'!B14:N22').copyTo(spreadsheet.getActiveRange(), SpreadsheetApp.CopyPasteType.PASTE_VALUES, false);
          spreadsheet.getRange('\'Calc Individual\'!B14:N22').copyTo(spreadsheet.getActiveRange(), SpreadsheetApp.CopyPasteType.PASTE_FORMAT, false);
          spreadsheet.getActiveSheet().getFilter().remove();
          spreadsheet.getDataRange().createFilter();
        };
